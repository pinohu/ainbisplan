## Selection rules

1. Select the archetype from the user’s dominant task, not the organization’s industry label.
2. Add secondary capabilities only where they change the component, content, risk, or workflow model.
3. Use clinical, legal, financial, and children’s extensions whenever harm or regulatory sensitivity warrants stricter rules.
4. Do not use an authority-site layout for an operational dashboard merely because the organization publishes content.
5. Do not use a dashboard aesthetic for public content merely because the content is generated from a database.
6. Every business receives a brand profile; not every business receives custom components.
7. A custom component must demonstrate that tokens, core components, composition, and archetype patterns cannot safely express the need.

## New-project intake

Every new initiative should record:

- audience;
- dominant jobs to be done;
- risk and trust threshold;
- primary conversion;
- content depth;
- operational complexity;
- geographic scope;
- accessibility needs beyond the shared minimum;
- primary archetype;
- secondary capabilities;
- brand-engine inputs;
- required domain review;
- implementation owner;
- launch and verification criteria.
